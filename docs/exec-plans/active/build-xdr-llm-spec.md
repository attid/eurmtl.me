# Build XDR LLM Spec

## Контекст
`GET /llm.txt` уже существует как машинная точка входа, но описание `POST /lab/build_xdr` слишком поверхностное. Агенту без браузера нужна полная спецификация по поддерживаемым операциям, полям, форматам значений и special cases.

## План изменений
1. [ ] Добавить в `templates/llm.txt` отдельный подробный раздел для `POST /lab/build_xdr`.
2. [ ] Описать верхнеуровневые поля payload и общие правила для `operations[]`.
3. [ ] Описать каждую поддерживаемую операцию из `services/stellar_client.py` и её поля.
4. [ ] Добавить короткие примеры payload для типового использования.

## Риски и открытые вопросы
- Описание должно совпадать с фактической реализацией `stellar_build_xdr`, а не с желаемым API.
- Нельзя описывать поля, которых код на самом деле не читает.

## Верификация
- Вручную сверить `templates/llm.txt` с `services/stellar_client.py::stellar_build_xdr`
- `UV_CACHE_DIR=/tmp/uv-cache just check-changed`
