# Golden Principles

These principles are intentionally stable and should rarely change.

1. Predictability over cleverness.
2. Mechanical verification over manual confidence.
3. Small diffs over broad refactors.
4. Contracts first: parse and validate, do not guess.
5. Architecture boundaries are product constraints, not suggestions.
6. Documentation is part of the runtime safety system.
7. If a rule cannot be checked, add a check or narrow the claim.
